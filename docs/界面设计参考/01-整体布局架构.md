# 整体布局架构

## 三段式终端布局

Claude CLI 采用经典的"三段式"终端布局，将界面划分为三个主要区域：

```
┌─────────────────────────────────────────────┐
│  上部：配置历史记录区                          │
│  (Configuration History)                     │
├─────────────────────────────────────────────┤
│  中部：对话上下文与实时状态区                    │
│  (Context & Status)                          │
│                                              │
│  * Worked for 44s                           │
│  ┌─────────────────────────────────────┐    │
│  │ 可以，且目前支持最多已确认的4个之外最多配置3个自定义 │    │
│  └─────────────────────────────────────┘    │
│  * Envisioning... (45s · thinking some more)│
│                                              │
├─────────────────────────────────────────────┤
│  下部：输入提示符与控制栏                        │
│  (Input & Controls)                          │
│  > _                                         │
│  ─────────────────────────────────────────── │
│  esc to interrupt                            │
└─────────────────────────────────────────────┘
```

## 区域功能说明

### 上部：配置历史记录区
- 显示用户已完成的配置操作流程
- 包含向导式交互的日志回显
- 展示菜单选择、表单填写、确认反馈等历史

### 中部：对话上下文与实时状态区
- **耗时统计**：显示 AI 处理时间（如 `* Worked for 44s`）
- **用户消息**：灰色高亮背景条，模拟消息气泡效果
- **实时状态**：黄色文字显示 AI 处理状态（如 `* Envisioning...`）
- **计时器**：精确到秒的耗时显示
- **子状态**：更细粒度的状态描述（如 `thinking some more`）

### 下部：输入提示符与控制栏
- **输入提示符**：`> ` 符号 + 绿色闪烁光标
- **分隔线**：灰色水平线，隔离历史内容与输入区
- **辅助提示**：灰色文字显示控制快捷键（如 `esc to interrupt`）

## 设计要点

1. **垂直流式堆叠**：按时间顺序和功能逻辑排列
2. **视觉层次清晰**：通过颜色和符号区分不同区域
3. **信息密度适中**：紧凑但不拥挤，便于快速扫描
4. **响应式布局**：适应不同终端宽度

## 适配 forge-cli 的考虑

- 考虑 forge-cli 的功能特点，可能需要增加更多状态区域
- 预留插件系统、工具调用等扩展区域的位置
- 保持布局的灵活性，支持未来功能扩展
