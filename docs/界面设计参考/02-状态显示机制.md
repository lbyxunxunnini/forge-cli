# 状态显示机制

## 实时状态指示器

Claude CLI 的核心状态显示组件是实时状态对话框，用于展示 AI 的处理状态。

### 状态显示格式
```
* Envisioning... (45s · thinking some more)
│  │           │  │   │
│  │           │  │   └── 子状态描述
│  │           │  └── 耗时计时器
│  │           └── 进行中标识
│  └── 状态动词
└── 系统消息标识
```

### 状态动词库
- **Envisioning** - 构思、规划阶段
- **Thinking** - 思考、分析阶段
- **Generating** - 生成内容阶段
- **Processing** - 处理任务阶段
- **Executing** - 执行操作阶段

### 子状态描述
- `thinking some more` - 深度思考中
- `analyzing code` - 分析代码中
- `writing solution` - 编写解决方案中
- `reviewing changes` - 审查变更中

## 计时器机制

### 计时器显示
- 精确到秒：`(45s)`
- 实时更新：每秒刷新
- 颜色区分：黄色/金色表示进行中

### 计时器作用
1. **透明度**：让用户知道 AI 在工作
2. **预期管理**：帮助用户预估等待时间
3. **焦虑缓解**：避免"假死"错觉

## 状态颜色编码

| 状态类型 | 颜色 | 示例 |
|---------|------|------|
| 进行中 | 黄色/金色 | `* Envisioning...` |
| 成功完成 | 绿色 | `✓ 已切换到 model-a` |
| 错误失败 | 红色 | `✗ 配置失败` |
| 辅助信息 | 灰色 | `* Worked for 44s` |

## 多层级状态显示

### 第一层：主状态
- 显示当前主要任务状态
- 使用动词 + 省略号格式
- 例如：`* Envisioning...`

### 第二层：耗时信息
- 显示任务已消耗时间
- 使用括号包裹
- 例如：`(45s)`

### 第三层：子状态描述
- 显示更细粒度的状态
- 使用 `·` 分隔符
- 例如：`thinking some more`

## 异步任务状态管理

### 状态转换流程
```
开始 → 处理中 → 完成/失败
  │       │
  │       └── 显示计时器和子状态
  │
  └── 显示初始状态
```

### 长时间任务处理
1. **实时更新**：每秒更新计时器
2. **状态细化**：提供更详细的子状态
3. **用户控制**：允许中断操作（ESC 键）

## forge-cli 适配建议

### 状态类型扩展
- **工具调用状态**：`* Calling tool... (2s)`
- **插件加载状态**：`* Loading plugin... (1s)`
- **配置读取状态**：`* Reading config... (0.5s)`
- **网络请求状态**：`* Fetching data... (3s)`

### 子状态细化
- 工具调用：`preparing arguments`
- 插件加载：`initializing module`
- 配置读取：`parsing YAML`
- 网络请求：`waiting for response`

### 计时器优化
- 支持毫秒级显示：`(1.5s)`
- 支持分钟级显示：`(1m 30s)`
- 支持进度百分比：`(45% · 30s)`
