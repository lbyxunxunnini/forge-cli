# 符号系统

## 视觉符号约定

Claude CLI 使用一套简洁的符号系统，通过视觉符号传递复杂信息，减少认知负担。

### 核心符号

| 符号 | 含义 | 使用场景 |
|------|------|----------|
| `>` | 输入点、选择 | 用户输入提示符、菜单选择 |
| `✓` | 成功、完成 | 操作成功、确认反馈 |
| `✗` | 失败、错误 | 操作失败、错误提示 |
| `*` | 系统消息、元数据 | 系统状态、耗时统计 |
| `?` | 询问、确认 | 确认操作、询问用户 |
| `!` | 警告、注意 | 警告消息、重要提示 |
| `·` | 分隔、连接 | 子状态分隔、列表连接 |
| `-` | 列表项 | 无序列表、选项列表 |
| `1.` | 序号 | 有序列表、菜单选项 |

## 符号使用规范

### 输入相关符号
```
> 用户输入提示符
│ 输入区域标识
└ 输入区域边界
```

### 状态相关符号
```
✓ 成功状态
✗ 失败状态
* 系统状态
? 询问状态
! 警告状态
```

### 列表相关符号
```
- 无序列表项
1. 有序列表项
· 分隔符
→ 指向符
```

## 符号组合模式

### 确认反馈模式
```
✓ 已添加，选择默认模型:
1. model-a
2. model-b
```

### 询问确认模式
```
这样 OK 吗?
```

### 状态显示模式
```
* Envisioning... (45s · thinking some more)
```

### 错误提示模式
```
✗ 配置失败：API Key 无效
```

## 符号认知优化

### 认知负担降低
1. **一致性**：相同符号在所有场景下含义一致
2. **简洁性**：符号简单易记，无歧义
3. **直观性**：符号含义直观，无需学习
4. **区分度**：不同符号视觉差异明显

### 符号学习曲线
- **即时理解**：`>` `✓` `✗` 等常见符号
- **快速学习**：`*` `?` `!` 等系统符号
- **深度掌握**：符号组合模式

## 符号扩展机制

## forge-cli 适配建议

### 扩展符号集
```
⚡ 快速操作
🔧 工具调用
📦 插件加载
🔗 链接、依赖
📊 数据、统计
🎯 目标、任务
```

### 符号使用场景
- **工具调用**：`🔧 Calling tool...`
- **插件加载**：`📦 Loading plugin...`
- **配置更新**：`⚙ Updating config...`
- **任务完成**：`🎯 Task completed`
