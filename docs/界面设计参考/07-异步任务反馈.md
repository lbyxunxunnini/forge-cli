# 异步任务反馈

## 长时间任务状态展示

Claude CLI 针对 AI 可能存在的长时间响应，设计了多层级的状态展示机制，有效缓解用户等待焦虑。

### 状态展示格式
```
* Envisioning... (45s · thinking some more)
│  │           │  │   │
│  │           │  │   └── 子状态描述
│  │           │  └── 耗时计时器
│  │           └── 进行中标识
│  └── 状态动词
└── 系统消息标识
```

## 三重反馈机制

### 第一重：颜色变化
- **绿色** → **黄色**：表示从等待状态进入处理状态
- **黄色**：持续显示，表示正在处理
- **黄色** → **绿色**：表示处理完成

### 第二重：动态计时器
- **实时更新**：每秒更新耗时显示
- **精确显示**：精确到秒 `(45s)`
- **持续显示**：处理过程中持续显示

### 第三重：文本描述
- **状态动词**：`Envisioning...` 表示处理阶段
- **子状态**：`thinking some more` 表示详细状态
- **进度提示**：可选显示处理进度

## 焦虑缓解策略

### 透明度设计
1. **状态可见**：用户知道 AI 在工作
2. **进度可知**：用户知道已消耗时间
3. **阶段明确**：用户知道当前处理阶段

### 预期管理
1. **时间预期**：通过计时器帮助用户预估时间
2. **阶段预期**：通过状态动词帮助用户理解阶段
3. **结果预期**：通过子状态帮助用户预期结果

### 控制感设计
1. **中断能力**：ESC 键中断处理
2. **取消能力**：取消长时间任务
3. **跳过能力**：跳过某些处理阶段

## 状态转换流程

### 正常流程
```
开始 → 处理中 → 完成
  │       │
  │       └── 显示计时器和子状态
  │
  └── 显示初始状态
```

### 错误流程
```
开始 → 处理中 → 错误
  │       │
  │       └── 显示错误信息
  │
  └── 显示初始状态
```

### 中断流程
```
开始 → 处理中 → 中断
  │       │
  │       └── 用户按 ESC 键
  │
  └── 显示初始状态
```

## 超时处理机制

### 超时检测
- **时间阈值**：设置最大处理时间
- **警告提示**：接近超时时发出警告
- **自动中断**：超时后自动中断处理

### 超时处理策略
1. **继续等待**：用户选择继续等待
2. **中断处理**：用户选择中断处理
3. **降级处理**：使用降级方案继续

### 超时消息格式
```
* 处理时间较长 (2m 30s)
是否继续等待?
1. 继续等待
2. 中断处理
> _
```
