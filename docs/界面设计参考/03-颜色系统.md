# 颜色系统

## 语义化配色方案

Claude CLI 使用语义化的颜色系统，通过颜色传达状态和信息类型。

### 主色调
- **亮绿色** (`#00FF00`) - 主交互色
- **纯黑色** (`#000000`) - 背景色
- **高对比度** - 确保可读性

### 颜色语义映射

| 颜色 | 语义 | 使用场景 |
|------|------|----------|
| 亮绿色 | 正常、成功、用户输入 | 输入提示符、成功消息、用户消息 |
| 黄色/金色 | 进行中、处理中 | AI 状态显示、计时器 |
| 灰色 | 辅助、背景、历史 | 耗时统计、辅助提示、历史记录 |
| 红色 | 错误、失败、警告 | 错误消息、失败状态 |
| 蓝色 | 信息、提示 | 信息性消息、帮助文本 |

## 状态对应颜色

### AI 处理状态
```
开始处理 → 黄色
处理中   → 黄色（持续）
处理完成 → 绿色
处理失败 → 红色
```

### 用户交互状态
```
等待输入 → 绿色（光标闪烁）
输入中   → 绿色（文本显示）
输入完成 → 绿色（确认显示）
```

### 系统消息状态
```
系统信息 → 灰色
成功消息 → 绿色
警告消息 → 黄色
错误消息 → 红色
```

## 对比度设计

### 对比度要求
- **最小对比度**：4.5:1（WCAG AA 标准）
- **推荐对比度**：7:1（WCAG AAA 标准）
- **终端适配**：考虑不同终端的颜色支持

### 颜色组合
- **绿底黑字**：高对比度，适合长时间使用
- **黑底绿字**：经典终端风格，护眼
- **灰底白字**：辅助信息，不干扰主内容

## 颜色使用规范

### 输入区域
- 提示符：亮绿色
- 光标：亮绿色（闪烁）
- 输入文本：亮绿色

### 状态区域
- 进行中状态：黄色/金色
- 成功状态：亮绿色
- 错误状态：红色
- 辅助信息：灰色

### 消息区域
- 用户消息：灰色背景 + 白色文字
- AI 消息：默认前景色
- 系统消息：灰色

## 终端兼容性

### 颜色支持检测
- 检测终端颜色支持能力
- 自动降级到基本颜色
- 提供无颜色模式选项

### 颜色方案适配
- 支持浅色/深色主题切换
- 支持自定义颜色配置
- 支持颜色盲友好模式

## forge-cli 适配建议

### 扩展颜色语义
- **插件状态**：紫色（加载中、已加载）
- **工具状态**：青色（调用中、已完成）
- **配置状态**：橙色（读取中、已更新）

### 颜色配置选项
```json
{
  "colors": {
    "primary": "#00FF00",
    "secondary": "#FFD700",
    "success": "#00FF00",
    "error": "#FF4444",
    "warning": "#FFD700",
    "info": "#888888",
    "background": "#000000"
  }
}
```

### 主题系统
- 内置主题：经典终端、现代深色、浅色
- 自定义主题：用户可配置颜色
- 主题切换：运行时切换主题
