# 文本排版规范

## 等宽字体使用

### 字体选择
- **主字体**：等宽字体（Monospaced Font）
- **推荐字体**：Menlo, Monaco, Consolas, Courier New
- **字体大小**：12-14px（根据终端设置）

### 等宽字体优势
1. **对齐精确**：字符宽度一致，便于对齐
2. **代码可读**：代码块显示清晰
3. **表格整齐**：表格内容对齐整齐
4. **缩进一致**：缩进层次清晰可见

### 字体配置
```json
{
  "font": {
    "family": "Menlo, Monaco, Consolas, monospace",
    "size": 13,
    "weight": "normal"
  }
}
```

## 缩进对齐规范

### 缩进层次
```
第一层：无缩进
  第二层：2空格缩进
    第三层：4空格缩进
      第四层：6空格缩进
```

### 对齐方式
- **左对齐**：文本内容左对齐
- **右对齐**：数字、日期等右对齐
- **居中对齐**：标题、表头等居中对齐

### 对齐示例
```
名称        类型        说明
─────────────────────────────
config      object     配置对象
name        string     名称
version     string     版本号
```

## 空行使用规范

### 空行分隔
- **段落分隔**：段落之间使用空行分隔
- **区域分隔**：不同区域之间使用空行分隔
- **逻辑分隔**：逻辑块之间使用空行分隔

### 空行示例
```
第一段内容

第二段内容

第三段内容
```

## 信息密度控制

### 密度原则
1. **紧凑但不拥挤**：信息密集但可读
2. **层次清晰**：通过缩进和空行区分层次
3. **重点突出**：重要信息视觉突出
4. **扫描友好**：便于快速扫描阅读

### 密度调整
- **高密度**：专业用户、详细信息
- **中密度**：普通用户、标准信息
- **低密度**：新手用户、简化信息

## 文本装饰规范

### 装饰符号
- **分隔线**：`─────────────────`
- **边框**：`┌───┐ │ └───┘`
- **列表符号**：`-` `*` `+`
- **序号**：`1.` `2.` `3.`

### 装饰使用
```
┌─────────────────────────────────────┐
│ 标题                                │
├─────────────────────────────────────┤
│ 内容区域                             │
│ - 列表项 1                          │
│ - 列表项 2                          │
└─────────────────────────────────────┘
```

## 文本高亮规范

### 高亮方式
- **颜色高亮**：使用颜色突出重要信息
- **背景高亮**：使用背景色突出信息
- **符号高亮**：使用符号标记重要信息

### 高亮示例
```
✓ 成功消息（绿色）
✗ 错误消息（红色）
! 警告消息（黄色）
* 系统消息（灰色）
```

## 文本截断规范

### 截断策略
- **末尾截断**：文本过长时末尾截断
- **中间截断**：文本过长时中间截断
- **智能截断**：根据内容智能截断

### 截断显示
```
这是一个很长的文本内容，需要截断显示...

```

## 文本换行规范

### 换行策略
- **自动换行**：根据终端宽度自动换行
- **手动换行**：用户指定换行位置
- **智能换行**：根据内容智能换行

### 换行示例
```
这是一段很长的文本内容，会根据终端宽度
自动换行显示，确保内容完整可读。
```
