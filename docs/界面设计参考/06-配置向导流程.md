# 配置向导流程

## 向导式交互模式

Claude CLI 采用向导式（Wizard-style）交互模式，通过分步引导用户完成复杂配置。

### 向导流程示例
```
选自定义: > 5
名称 > my-provider
Base URL > https://xxx.com/v1
API Key > sk-xxx
模型（逗号分隔）> model-a, model-b
✓ 已添加，选择默认模型:
1. model-a
2. model-b
已切换到 my-provider / model-a
这样 OK 吗?
```

## 向导设计原则

### 分步引导
- **一次一个字段**：每步只询问一个配置项
- **渐进式披露**：逐步展示复杂配置
- **上下文保持**：保持配置流程的上下文

### 输入引导
- **提示符引导**：使用 `>` 符号引导输入
- **格式提示**：提供输入格式示例
- **默认值**：提供合理的默认值

### 反馈确认
- **即时反馈**：每步完成后立即反馈
- **成功确认**：使用 `✓` 符号确认成功
- **错误提示**：明确指出错误和修复方法

## 向导状态管理

### 状态类型
- **开始状态**：向导启动
- **输入状态**：等待用户输入
- **验证状态**：验证输入内容
- **处理状态**：处理配置请求
- **完成状态**：配置完成
- **错误状态**：配置失败

### 状态转换
```
开始 → 输入 → 验证 → 处理 → 完成
              ↓
           错误 → 输入
```

### 状态显示
- **输入状态**：绿色提示符
- **验证状态**：黄色状态显示
- **处理状态**：黄色计时器
- **完成状态**：绿色确认消息
- **错误状态**：红色错误消息

## 向导错误处理

### 输入验证
- **格式验证**：检查输入格式是否正确
- **范围验证**：检查输入值是否在有效范围内
- **唯一性验证**：检查输入值是否唯一

### 错误恢复
- **即时提示**：立即指出错误
- **修复建议**：提供修复建议
- **重新输入**：允许重新输入
- **跳过选项**：提供跳过选项

### 错误消息格式
```
✗ 错误：API Key 格式不正确
请重新输入 API Key:
> _
```

## 向导流程控制

### 流程导航
- **前进**：继续下一步
- **后退**：返回上一步
- **跳过**：跳过当前步骤
- **取消**：取消整个流程

### 流程分支
- **条件分支**：根据输入选择不同路径
- **动态步骤**：根据上下文动态显示步骤
- **可选步骤**：标记可选配置项

### 流程保存
- **自动保存**：每步完成后自动保存
- **手动保存**：提供手动保存选项
- **恢复功能**：支持恢复未完成的配置
