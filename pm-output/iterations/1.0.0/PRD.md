# 产品需求文档

## 1. 产品概览
- 产品名称：Forge CLI
- 版本：1.0.0
- 产品类型：B 端开发者工具
- 目标用户：开发者（和 Claude Code 用户重叠）
- 产品目标：解决 Claude Code 频繁确认、缓存命中率低、记忆无法跨会话传递的痛点
- MVP 边界：权限自由 + 缓存优化 + 记忆导出

## 2. 问题与机会
- 用户痛点：
  1. 权限自由：离开电脑 10 分钟就卡在确认，切一次确认后无记忆下次还需要确认
  2. 缓存优化：长对话后响应变慢，需要更高缓存命中率节省成本
  3. 记忆导出：长下文满了需要切会话，切会话前内容要重新描述
- 当前替代方案：
  - Codex：权限自由，但缓存差、不能接三方模型
  - Claude Code：能接三方模型，但权限不自由
- 为什么值得做：有真实场景支撑，痛点发生频率高、痛感强
- 证据状态：用户自述痛点，有具体使用场景
- 差异化：权限自由 + 缓存优化 + 记忆导出，三合一解决方案

## 3. 用户与场景
| 用户 / 角色 | 场景 | 目标 | 痛点 | 成功信号 |
|-------------|------|------|------|----------|
| 开发者 | 日常开发任务（写功能、改代码、跑测试） | 提高开发效率，减少中断 | 权限频繁确认、缓存命中率低、记忆无法跨会话 | 一天内不需要手动确认，缓存命中率 95%+，记忆可跨会话传递 |

## 4. 范围
### 本期范围
- 权限自由：本项目目录增改查读不需要确认，删除需要确认，外部第一次需要确认
- 缓存优化：目标 95% 以上甚至 99% 以上命中率
- 记忆导出：`/out-task` 命令，导出对话+ID，存储在 `.forge` 独立文件夹

### 不做范围
- 多模型适配（v2）
- 工作流系统
- 插件系统

### 后续范围
- 多模型适配：支持思考能力开启/关闭、高低中控制等模型特殊能力
- 工作流系统：已有功能，后续迭代优化
- 插件系统：已有功能，后续迭代优化

## 5. 功能需求
| ID | 功能 | 用户故事 | 优先级 | 业务规则 | 验收标准 |
|----|------|----------|--------|----------|----------|
| F1 | 权限自由 | 作为开发者，我希望离开电脑时任务不卡住，以便提高效率 | P0 | 本项目目录增改查读不需要确认，删除需要确认，外部第一次需要确认，确认后存入项目记忆 | 已记忆和项目内除了删除场景，一天内不需要手动确认 |
| F2 | 缓存优化 | 作为开发者，我希望长对话后响应不变慢，以便节省成本 | P0 | 缓存命中率目标 95% 以上甚至 99% 以上 | 缓存命中率达到 95% 以上 |
| F3 | 记忆导出 | 作为开发者，我希望一键导出记忆，以便在新会话中继续对话 | P0 | `/out-task` 命令，导出对话+ID，存储在 `.forge` 独立文件夹，成功后自动显示路径和 ID | 导出可以被下一个会话识别且没有丢失内容 |

## 6. 状态与边界
| 对象 / 流程 | 状态 | 触发条件 | 系统行为 | 边界情况 |
|-------------|------|----------|----------|----------|
| 权限确认 | 待确认 / 已同意 / 已同意并记住 | 用户执行操作 | 显示确认提示 `[Forge] 允许读取 /path/to/file? [1/2/3]` | 权限文件损坏或不存在时，默认全部需要确认 |
| 记忆导出 | 导出中 / 导出成功 / 导出失败 | 用户执行 `/out-task` | 导出对话+ID 到 `.forge` 独立文件夹 | 导出失败时（磁盘满、权限不足），提示用户 |
| 缓存状态 | 缓存命中 / 缓存未命中 | 用户发送消息 | 缓存命中率优化 | 缓存失效条件（文件变化、时间过期） |

## 7. 页面与组件交互要求
| 页面 / 组件 | 产品目的 | 用户操作 | 前置条件 | 系统反馈 | 异常处理 | 验收标准 |
|-------------|----------|----------|----------|----------|----------|----------|
| 权限确认提示 | 让用户控制权限 | 输入 1/2/3 | 用户执行需要权限的操作 | `[Forge] 允许读取 /path/to/file? [1/2/3]` | 权限文件损坏时默认全部需要确认 | 选项 1/2/3 可用，选择后执行相应操作 |
| 记忆导出命令 | 一键导出记忆 | 输入 `/out-task` | 有对话历史 | `[Forge] 记忆已导出: .forge/memory/abc123.md (ID: abc123)` | 导出失败时提示用户 | 导出成功，显示路径和 ID |

## 8. 核心按钮细节
| 按钮 | 所在页面 / 组件 | 文案 | 启用条件 | 禁用原因展示 | 点击后行为 | 加载 / 成功 / 失败反馈 | 防重复提交 | 埋点 / 验收 |
|------|-----------------|------|----------|--------------|------------|------------------------|--------------|-------------|
| 权限确认选项 1 | 权限确认提示 | 1（不同意） | 有权限操作 | 无 | 拒绝操作 | 无 | 无 | 验收：选择后操作被拒绝 |
| 权限确认选项 2 | 权限确认提示 | 2（同意一次） | 有权限操作 | 无 | 同意操作，不保存记忆 | 无 | 无 | 验收：选择后操作执行，记忆不保存 |
| 权限确认选项 3 | 权限确认提示 | 3（同意并记住） | 有权限操作 | 无 | 同意操作，保存记忆 | 无 | 无 | 验收：选择后操作执行，记忆保存 |
| 记忆导出命令 | CLI 命令 | /out-task | 有对话历史 | 无 | 导出对话+ID | 成功：显示路径和 ID；失败：提示用户 | 无 | 验收：导出成功，显示路径和 ID |

## 9. 非功能需求
- 性能：缓存命中率 95% 以上甚至 99% 以上
- 可访问性：CLI 工具，无特殊要求
- 安全 / 隐私：权限记忆存储在 `.forge/permissions.json`，仅项目内使用
- 兼容性：Node.js 18+
- 多语言 / 地区：中文界面

| 实体 / 内容 | 来源 | 字段 | 创建 / 更新规则 | 备注 |
