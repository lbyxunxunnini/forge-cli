# Existing Project Consulting

存量项目场景下的顾问式提问规则，通用提问格式见 [advisor_collaboration.md](advisor_collaboration.md)。

## 适用场景

- 迭代中项目首次扫描
- 存量项目内的新页面或模块扩展
- 复用判断
- 规则冲突
- 改动边界不清
- 重构、迁移、review

## 不能直接做的五种情况

1. 多种主流写法并存，无法直接判断应该跟哪一套
2. 复用旧代码和新写都合理，但长期维护成本不同
3. 改动可能触碰公共组件、共享状态、公共样式或主路由
4. 规则文件、guardrails 与实际代码相互冲突
5. 用户目标明确，但落点模块或页面边界不清

## 存量项目特有的提问约束

- 先给 2-4 行判断摘要（基于项目扫描结果）
- 不要把每个低风险细节都变成问题，只问会影响实现落点或长期维护的决策
