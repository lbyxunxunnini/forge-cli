# Role Gate Matrix

> **P0 强制规则**：每个角色放行/完成时必须输出对应 Mandatory Checklist 结构化 YAML 块（定义在各角色文件中），由 `scripts/validate_checklist.py` 校验。未输出合规 YAML 块不得宣布该角色结论。

## 需求分析师

- 进入：从需求 / PRD / 页面规划开始
- 停机：关键逻辑未补全
- 放行：Mandatory Checklist 全部 `[x]` 或 `[-]` + 用户确认状态允许进入下一角色
- 建议预算：`L2 2-4` 问，`L3 4-7` 问，`L4 6-10` 问分阶段执行

## UI 设计师

- 进入：需求已确认，且具备可靠设计输入
- 停机：视觉输入不可读，或多种拆法都合理
- 新项目共创：先过方向确认，再过结构草案确认，才允许展开完整页面结构包
- 放行：Mandatory Checklist 全部 `[x]` 或 `[-]` + 结构树、组件边界、交互边界足够支撑架构决策
- 建议预算：`L2 1-2` 次确认，`L3 2-4` 次确认，`L4 3-5` 次确认

## 架构设计师

- 进入：需求和 UI 已足够支持模块归属、文件结构、状态和命名决策
- 停机：高风险公共抽象、不可逆改动、规则冲突
- 放行：Mandatory Checklist 全部 `[x]` 或 `[-]` + guardrails / 结构方案 / 复用策略明确
- 预算原则：只在存在模块归属、复用、状态管理或命名决策时介入，不为仪式感发言

## 页面工程师

- 进入：已有结构化决策，或属于明确轻量任务
- 停机：实现中发现需求 / UI / 架构缺口或技术风险
- 放行：Mandatory Checklist 全部 `[x]` 或 `[-]` + 代码、验证、边界态、规则一致性通过
- 预算原则：轻量任务保持简短路径并豁免最强写前确认等待；完整流程中只问阻塞实现的缺口

## 验证工程师

- 进入：S5 验证阶段
- 停机：验证发现需求覆盖缺口或回归问题
- 放行：Mandatory Checklist 全部 `[x]` 或 `[-]` + 风险已收口
- 预算原则：只在验证结果与冻结约束有偏差时发声
