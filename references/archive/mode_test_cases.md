# Flutter Forge Reference - 模式测试用例

这个文件给每个模式一条最小测试用例，方便在改协议后快速回归：

- 路由是否正确
- 模式输出是否正确
- 阶段日志是否合理
- 是否按预期自动退出
- 大任务是否正确显示子 agent / 并行提效

## 使用方法

每条用例至少验证 4 件事：

1. 用户输入是否命中正确模式
2. `[f-forge]` 模式日志是否正确
3. 阶段日志和角色日志是否符合该模式最小输出
4. 本轮任务完成后是否自动退出
5. 若输入为 `ff-fast`，是否保持轻量 / 中等快速路径，只有明确风险才升级
6. 若输入为 `ff-a`，是否先输出全自动日志并在结束时给出全自动摘要

---

## 1. 直通模式

### 输入

```text
ff- 帮我改一下 README 里的安装说明
```

### 预期输出

```text
[f-forge] 直通模式：快速处理
[f-forge] 本轮完成：任务已完成，已自动退出 flutter-forge
```

### 核心断言

- 不展开阶段日志
- 不拉起需求分析师 / UI 设计师 / 架构设计师
- 处理文档类任务后直接退出

---

## 2. 轻量任务

### 输入

```text
ff- 把首页购买按钮点击后跳到会员购买页
```

### 预期输出

```text
[f-forge] 页面工程师：轻量任务，直接执行
[f-forge] 页面工程师：已完成修改并完成基本验证
[f-forge] 本轮完成：任务已完成，已自动退出 flutter-forge
```

### 核心断言

- 只输出开始和结束两条主要日志
- 不进入 `S1/S2`
- 不要求专项判断

---

## 3. 中等任务

### 输入

```text
ff- 订单页加 3 个筛选条件，先看相似实现再改
```

### 预期输出

```text
[f-forge] 页面工程师：中等任务，先扫描后执行
[f-forge] 页面工程师：先看相似实现和当前页面结构，再决定直接修改还是升级流程
[f-forge] 本轮完成：任务已完成，已自动退出 flutter-forge
```

### 核心断言

- 页面工程师先扫描
- 默认不进入 `S1`
- 只有发现需求缺口/结构风险时才升级

---

## 4. UI 优化

### 输入

```text
ff- 优化个人中心页面视觉层级和卡片布局
```

### 预期输出

```text
[f-forge] 模式：UI 优化
[f-forge] 阶段：S2 方案确认
[f-forge] UI 设计师：现有问题在视觉层级和卡片主次关系。
[f-forge] 页面工程师：将按该方案调整现有页面，不改业务逻辑。
[f-forge] 本轮完成：已完成个人中心视觉层级和卡片布局优化；修改了页面样式、间距和主次关系；实现了不改业务逻辑的样式收口。
[f-forge] 本轮完成：任务已完成，是否需要更新规则卡
- 否：任务结束已自动退出 flutter-forge
```

### 核心断言

- 先 UI 判断，再进入实现
- 如果缺少会影响放行的 UI 输入，应先向用户补要设计图、参考图或文字化 UI 描述
- 默认不进入 `S1`
- 若发现业务闭环变化，应升级为 `功能开发` 或 `页面开发`

---

## 5. 架构级任务

### 输入

```text
ff- 帮我做包体积优化
```

### 预期输出

```text
[f-forge] 模式：架构级任务
[f-forge] 阶段：S2 方案确认
[f-forge] 架构设计师：先冻结依赖、资源和构建参数优化范围，再进入实现。
[f-forge] 页面工程师：按冻结后的边界做改动并补验证。
[f-forge] 本轮完成：已完成包体积优化；修改了依赖、资源和构建参数；实现了发布产物瘦身。
[f-forge] 本轮完成：任务已完成，是否需要更新规则卡
- 是：更新规则卡
任务结束，已自动退出 flutter-forge
```

### 核心断言

- 默认从 `S2` 开始
- 重点是边界、策略、迁移或优化，不是新页面/业务闭环
- 若同时缺失业务约束，允许先插入 `S1`

## 6. 功能开发

### 输入

```text
ff- 当客态用户点击邀请 h5 中“录制”按钮拉起 app 后，按弹窗优先级、授权弹窗、有无音色和提示栏逻辑完成整条交互
```

### 预期输出

```text
[f-forge] 模式：功能开发
- 升级原因：需求涉及邀请唤起、弹窗优先级、授权弹窗、录制流程和提示栏联动，属于跨页面状态与业务闭环开发
[f-forge] 阶段：S1 需求确认
[f-forge] 需求分析师：先冻结拉起条件、优先级、有无音色双分支、录制回流和提示栏消失时机。
[f-forge] 阶段：S2 方案确认
[f-forge] 架构设计师：冻结深链承接、状态缓存、授权回写和提示栏回流链路。
[f-forge] 本轮完成：已完成邀请承接、授权弹窗和提示栏联动；修改了状态流转、回流链路和验证逻辑；实现了完整业务闭环。
[f-forge] 本轮完成：任务已完成，是否需要更新规则卡
- 是：更新规则卡
任务结束，已自动退出 flutter-forge
```

### 核心断言

- 不应误判为 `架构级任务`
- 不应因为没有新增页面就降成轻量或中等
- 核心是业务闭环和跨页面状态联动

## 7. 页面开发

### 输入

```text
ff- 新建一个商品详情页
```

### 预期输出

```text
[f-forge] 模式：页面开发
- 升级原因：新增页面，涉及页面结构、状态接入和路由组织
[f-forge] 阶段：S1 需求确认
[f-forge] 需求分析师：先补齐核心区块、返回路径和首屏目标。
[f-forge] 阶段：S2 方案确认
[f-forge] UI 设计师：确定图片区、信息区和底部操作栏结构。
[f-forge] 本轮完成：已完成商品详情页；修改了页面结构、状态接入和路由组织；实现了详情展示与底部操作区。
[f-forge] 本轮完成：任务已完成，是否需要更新规则卡
- 是：更新规则卡

```

### 核心断言

- 核心是页面/模块新增
- 应使用 `页面开发`，不是 `功能开发`
- 通常需要页面结构与路由接入
- 如果关键 UI 输入不足，应先向用户补要设计图、参考图或文字化 UI 描述

## 8. 新项目共创

### 输入

```text
ff- 我想做一个新的 Flutter 项目，先共创一下，只定项目架构和登录页
```

### 预期输出

```text
[f-forge] 模式：新项目共创
- 升级原因：当前是从 0 到 1 的新项目起步，需要先冻结工程结构和首批页面范围
[f-forge] 阶段：C0 想法收口
[f-forge] 需求分析师：先收口业务目标和第一版范围。
[f-forge] 阶段：C2 工程定型
[f-forge] 架构设计师：冻结目录、状态管理、路由和网络层方案。
[f-forge] 本轮完成：已完成新项目首批范围和工程方案共创；修改了目录、状态管理和路由约束；实现了首轮起步方案冻结。
[f-forge] 本轮完成：任务已完成，是否需要更新规则卡
- 是：更新规则卡
任务结束，已自动退出 flutter-forge
```

### 核心断言

- 不直接进入实现
- 必须经过 `C0-C3`
- 首批范围冻结后才能进入执行轨道

```text

```

### 预期输出

```text

- 升级原因：新增业务模块，涉及页面结构、模块边界、状态归属和路由接入
[f-forge] 阶段：S3 拆包冻结
[f-forge] 架构设计师：arch-agent 已冻结 4 个工作包，允许并行实现。
[f-forge] 阶段：S4 实现中
[f-forge] 页面工程师：impl-agent-1 正在处理权益页；impl-agent-2 正在处理任务页；impl-agent-3 正在处理积分记录。
[f-forge] 页面工程师：impl-agent-1 处理完成：已完成权益页结构和入口区块。
[f-forge] 页面工程师：impl-agent-2 处理完成：已完成任务页列表和状态切换。
[f-forge] 页面工程师：impl-agent-3 处理完成：已完成积分记录列表和分页承接。
[f-forge] 阶段：S5 验证中
```

## 10. 全自动页面开发

### 输入

```text
ff-a 新建一个商品详情页，包含轮播图、价格、规格选择和底部购买按钮，缺少的部分按推荐方案自动做完
```

### 预期输出

```text
[f-forge] 全自动：已启用 ff-a，非阻塞缺口将采用推荐方案推进；安全、不可逆或高风险架构决策才中断确认。
[f-forge] 模式：页面开发
- 升级原因：新增页面，涉及页面结构、规格选择状态、底部购买入口和路由接入
[f-forge] 阶段：S1 需求确认
[f-forge] 需求分析师：规格联动规则未指定，自动采用推荐方案：选择规格后同步刷新价格、库存和购买按钮可用状态。
[f-forge] 阶段：S2 方案确认
[f-forge] UI 设计师：未提供设计图，自动采用推荐方案：沿用项目现有详情页卡片风格。
[f-forge] 架构设计师：未指定状态管理，自动沿用规则卡或项目主流状态管理方案。
[f-forge] 阶段：S4 实现中
[f-forge] 阶段：S5 验证中
[f-forge] 全自动摘要：本轮自动采用 3 项推荐方案：规格联动价格库存；详情页卡片风格；项目主流状态管理接入。
[f-forge] 本轮完成：任务已完成，是否需要更新规则卡
- 是：更新规则卡
任务结束，已自动退出 flutter-forge
```

### 核心断言

- `ff-a` 是全自动策略，不是独立模式
- 必须先输出全自动启动日志，再输出真实任务模式
- 普通缺口不向用户追问，必须输出自动采用的推荐方案
- 必须保留 S1/S2/S4/S5 和验证
- 高风险不可逆事项仍需中断确认

## 11. 快速中等任务

### 输入

```text
ff-fast 订单页加 3 个筛选条件，先快速看相似实现再改
```

### 预期输出

```text
[f-forge] 页面工程师：ff-fast 快速策略，轻量优先，发现架构风险再升级。
[f-forge] 页面工程师：中等任务，先扫描后执行
[f-forge] 页面工程师：先用 project snapshot 和 1-3 个相似实现定位筛选逻辑，确认不涉及路由、状态管理或组件边界后直接修改。
[f-forge] 页面工程师：已按 ff-fast 完成修改并完成最小验证。
[f-forge] 本轮完成：任务已完成，已自动退出 flutter-forge
```

### 核心断言

- `ff-fast` 是快速策略，不是全自动策略
- 默认保持轻量 / 中等路径
- 不因为局部筛选条件直接进入完整流程
- 发现路由、状态管理、组件边界风险时必须升级并说明原因
