# Flutter Forge Reference - 代理输出格式

这个文件不再定义“角色顺序交接文档”，而是定义各专项代理在返回给 `controller` 时应携带的最小结构。

## requirement-analyst 返回格式

```markdown
## 任务语义
- 页面型 / 功能型 / 架构型

## 需求摘要
- 业务目标：...
- 页面目标 / 功能目标 / 架构目标：...
- 主流程 / 用户路径：...

## 关键约束
- 必须覆盖的状态：...
- 核心交互：...

## 待确认项
- [ ] ...

## 放行结论
- 是否允许进入 S2：是 / 否
```

如果是大改版或长文档输入，再补：

```markdown
## 需求冻结摘要包
- 本轮目标：...
- 当前范围：...
- 非目标：...
- 关键分支：...
- 当前未决项：...
```

## ui-agent 返回格式

```markdown
## UI 来源
- 真实视觉输入 / 用户文字描述 / 结构推断

## 结构判断
- 页面结构：...
- 区块顺序：...
- 组件边界：...

## 阻塞项
- [ ] ...

## 放行结论
- 是否允许进入实现：是 / 否
```

如果 UI 文档过长，再补：

```markdown
## UI 冻结摘要包
- 当前处理区块：...
- 关键布局：...
- 关键交互：...
- 待补 UI 输入：...
```

## arch-agent 返回格式

```markdown
## 结构决策
- 模块归属：...
- 状态归属：...
- 路由接入：...

## 冻结约束
- ...

## 复用策略
- ...

## 风险
- ...

## 放行结论
- 是否允许进入实现：是 / 否
```

如属大任务，再补：

```markdown
## 工作包
- P1: ...
- P2: ...
```

如果是大改版或跨模块任务，再补：

```markdown
## 架构冻结摘要包
- 当前落点：...
- 共享约束：...
- write scope：...
- 禁止改动区域：...
```

## impl-agent 返回格式

```markdown
## 当前动作
- ...

## 实现范围
- 文件 / 目录：...

## 阻塞或风险
- ...

## 验证结果
- ...
```

## verify-agent 返回格式

```markdown
## 验证范围
- ...

## 验证结果
- ...

## 风险汇总
- ...

## 放行结论
- 是否允许进入 S6：是 / 否
```

## 总规则

- 这些格式服务的是 `controller` 汇总，不是给用户看的完整仪式文档
- 只有主控认为需要时，才把其中一部分转成对用户可见的 `[f-forge]` 输出
- 不要把交接格式误当成必须逐段完整展示给用户
- 遇到超长原始输入时，优先在角色内部压缩成“阶段摘要包”，再交给下游
- 多个角色回传待确认项时，由 `controller` 合并成一次最小提问，不直接逐角色追问用户
